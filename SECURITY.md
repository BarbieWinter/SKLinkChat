# Security Policy

## Supported Versions

当前公开仓库只维护 `main` 分支。

## Reporting a Vulnerability

请不要把安全问题公开提交到 GitHub Issues。

如果你发现以下问题，请通过项目维护者指定的私密渠道报告：

- API key、数据库密码、私钥或访问 token 泄露
- 鉴权绕过
- 管理权限绕过
- WebSocket 会话串线
- 用户数据泄露
- 可导致服务不可用的漏洞

报告中请尽量包含：

- 问题描述
- 影响范围
- 复现步骤
- 你观察到的实际结果
- 你认为安全的修复建议，如有

维护者确认后会优先处理，并在修复完成后更新公开说明。
